# Тема 4: Бинарный поиск

> **Главная идея темы:** на **отсортированном** пространстве ответа делим область пополам за O(log n). Применим не только к массиву, но и к диапазону «ответа» (binary search on answer).

---

## Когда применять

1. Массив отсортирован, нужен поиск/граница.
2. Монотонная функция `check(x)` — можем отсортировать пространство ответов.
3. Просится O(log n) вместо O(n).

## Паттерны

- **Классический** — `while left <= right`, сужаем по сравнению с `target`.
- **Нижняя/верхняя граница** — поиск первого/последнего вхождения.
- **Бинарный поиск по ответу** — `check(mid)` → двигаем границы ответа (Koko, Split Array).
- **Поиск в rotated** — доп. проверка, какая половина отсортирована.
